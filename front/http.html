    # HTTP

    HTTP 超文本传输协议

    # HTTP缓存

    HTTP缓存可以分为两大类，强缓存和协商缓存。

    缓存的优点

    • 节省宽度流量
    • 减少服务器的负担，
    • 加快网页加载的速度

    # 强制缓存

    强制缓存，在缓存过期前，从缓存中获取，不会从服务器发送请求。状态码为200

    • from memory cache

    不请求网络资源，资源在内存当中，一般脚本、字体、图片会存在内存当中。浏览器关闭后，数据不存在

    • from disk cache

    不请求网络资源，在磁盘当中，一般非脚本会存在内存当中，如css等。 浏览器关闭后仍然存在

    1. Expires

    ('Expires', Sat, 25 Apr 2020 13:32:03 GMT)

    response header设置Expires，缓存过期时间GMT时间。

    1. Cache-Control

    ('Cache-Control', 'public,max-age=120')

    private, public max-age, must-revalidate

    缓存过期的相对时间，几分钟内过期。

    # 协商缓存

    第一次请求数据时，服务器将缓存标识响应给客户端，客户端备份在缓存中，再次请求时，会将缓存中的标示发给服务器，服务器根据标示判断。若未失效，返回304，浏览器直接使用缓存数据。

    1. Last-Modified

    Last-Modified 服务器在响应请求时，会告诉浏览器资源的最后修改时间

    if-Modified-Since: 浏览器再次请求服务器的时候，请求头会包含此字段，后面跟着在缓存中获得的最后修改时间

    1. Etag

    Etag 服务器响应请求时，通过此字段告诉浏览器当前资源在服务器生成的唯一标识（生成规则由服务器决定）

    If-None-Match： 再次请求服务器时，浏览器的请求报文头部会包含此字段，后面的值为在缓存中获取的标识。服务器接收到次报文后发现If-None-Match则与被请求资源的唯一标识进行对比。

    # HTTP 请求方法

    • GET

    用于请求资源获取数据，可以缓存

    • POST

    用于提交表单或者上传文件

    • HEAD

    与get请求类似，返回的相应中没有具体的内容，用于获取报头

    • OPTIONS

    获取服务器支持的HTTP请求方法。跨域请求时，复杂请求（自定义headers）在发送真正请求前发送Options请求。

    • DELETE

    请求服务器器删除Request-URI所标识的资源

    • PUT

    向指定资源位置上传其最新内容，覆盖

    • TRACE

    回显服务器收到的请求，主要用于测试或诊断

    • CONNECT

    建立一个到由目标资源标识的服务器的隧道。

    • PATCH

    PATCH 方法是非幂等的，连续多个的相同请求会产生不同的效果

    # 简单请求

    • 请求方法
      • GET
      • HEAD
      • POST
    • HEAD只能以下
      • Accept
      • Accept-Language
      • Content-Language
      • Content-Type
      • DPR
      • Downlink
      • Save-Data
      • Viewport-Width
      • Width
    • cotent-type为以下
      • text/plain
      • multipart/form-data
      • application/x-www-form-urlencoded
    • 请求中的任意XMLHttpRequestUpload 对象均没有注册任何事件监听器；XMLHttpRequestUpload 对象可以使用 XMLHttpRequest.upload 属性访问。
    • 请求中没有使用 ReadableStream 对象。

    # 预检请求

    需预检的请求”要求必须首先使用 OPTIONS 方法发起一个预检请求到服务器，以获知服务器是否允许该实际请求。"预检请求“的使用，可以避免跨域请求对服务器的用户数据产生未预期的影响。

    Access-Control-Allow-Origin(必须) Access-Control-Allow-Credentials(可选) Access-Control-Expose-Headers(可选)

    Access-Control-Request-Method Access-Control-Request-Headers Access-Control-Allow-Origin(必须) Access-Control-Allow-Methods (必须) Access-Control-Allow-Headers Access-Control-Allow-Credentials(可选) Access-Control-Max-Age(可选)
