      # WebSocket

      WebSocket实现了双向通信，服务端可以主动发送信息给客户端。在WebSocket中，浏览器和服务器只需要完成一次握手，两者之间就直接可以创建持久性的连接，并进行双向数据传输，客户端和服务器之间的数据交换变得更加简单。

      • web-sockets标准
      • mozilla文档

      # WebSocket使用场景

      • 弹幕
      • 在线协同
      • 社交聊天
      • 股票数据更新
      • 实时更新的数据
      • 位置信息更新

      # WebSocket接口

      enum BinaryType { "blob", "arraybuffer" };
[Exposed=(Window,Worker)]
interface WebSocket : EventTarget {
  constructor(USVString url, optional (DOMString or sequence<DOMString>) protocols = []);

  readonly attribute USVString url;

  // ready state状态
  const unsigned short CONNECTING = 0;
  const unsigned short OPEN = 1;
  const unsigned short CLOSING = 2;
  const unsigned short CLOSED = 3;
  readonly attribute unsigned short readyState;
  readonly attribute unsigned long long bufferedAmount;

  // networking
  attribute EventHandler onopen;
  attribute EventHandler onerror;
  attribute EventHandler onclose;
  readonly attribute DOMString extensions;
  readonly attribute DOMString protocol;
  void close(optional [Clamp] unsigned short code, optional USVString reason);

  // messaging
  attribute EventHandler onmessage;
  attribute BinaryType binaryType;
  void send(USVString data);
  void send(Blob data);
  void send(ArrayBuffer data);
  void send(ArrayBufferView data);
};

      # WebSocket构造函数

const socket = new WebSocket(url, [,protocols])

      url参数的格式

      • ws-URI = "ws:" "//" host [ ":" port ] path [ "?" query ]
      • wss-URI = "wss:" "//" host [ ":" port ] path [ "?" query ]

      # 属性

      • url 只读 数创建WebSocket实例对象时URL的绝对路径。

      • readyState 只读

        当前连接的状态

        • WebSocket.CONNECTING = 0 尚未建立连接
        • WebSocket.OPRN = 1 连接已建立，可以进行通信
        • WebSocket.CLOSING = 2 连接正在进行关闭握手，或者已调用close（）方法。
        • WebSocket.CLOSED = 3 连接已关闭或无法打开

      • bufferedAmount 只读

      • onopen 打开与WebSocket的连接时触发的回掉函数

      • onerror 当由于错误而关闭与WebSocket的连接（例如无法发送某些数据）时触发。

      • onclose 与WebSocket的连接关闭时触发

      • extensions 只读

      • protocol 只读

      • onmessage 通过WebSocket接收数据时触发

      • binaryType websocket连接所传输二进制数据的类型

      # 方法

      # close(code, reason)

      code为关闭的状态码,默认为1005

      reason为关闭的原因字符串

      # send(data)

      • data是字符串
      • data是Blob对象
      • data是ArrayBuffer对象
      • data是ArrayBufferView

      # 关闭连接

      客户端和服务端都可主动关闭

      # WebSocket库

      • socket.io 一个基于长轮询/WebSocket的Node.js第三方传输协议。
      • ws 一个流行的WebSocket客户端和服务器 Node.js库。
