# Node多版本管理

经常会遇到这样的问题，不同的项目需要用到不同版本的Node.js版本，所以电脑上要按照不同版本的Node版本，同时存在，方便切换。目前主要由两个方案，NVM（Node Version Manangerment） 和 n

# 1.NVM

NVM Node Version Manangerment

# 安装和更新nvm

curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.35.3/install.sh | bash
1

或者

wget -qO- https://raw.githubusercontent.com/creationix/nvm/v0.35.3/install.sh | bash
1

# 验证是否成功

关闭终端重新打开终端，输入nvm验证是否成功。

若提示"Node Version Manager",表示已经按照成功。

若提示"-bash: nvm: command not found"，表示还需要继续操作

export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"  # This loads nvm
[ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"  # This loads nvm bash_completion
1
2
3

然后继续执行nvm，确保已经按照成功

# 安装稳定版Node

nvm install stable
1

# 安装指定版本

nvm install <version>
1

# 删除已安装的指定版本

nvm uninstall <version>
1

# 切换成指定的版本

nvm use <version>
1

# 列出所有安装的版本

nvm ls
1
2

# 显示Node官方所有版本

nvm ls-remote
1

# 2.n

github 交互式管理Node.js版本

# 安装n

npm install -g n
1

# 安装特定版本的node

n 10.15.3
1

# 显示当前电脑所有已安装的node版本

n
1

用上下键可以切换当前需要的版本

# 安装官方最新的版本

n latest
1

# 安装最新的长期支持版LTS

n lts
1

# 删除特定版本

n rm 10.15.3
1

# 删除其他版本除了当前版本

n prune
1

# 帮助

n --help
// 或者
n -h
1
