          # Object.defineProperty()缺点

          无法监听数组的变化，

          要对每个对象的每个属性惊喜遍历，如果属性值也是对象，那需要深度遍历。

          Proxy直接代理整个对象， Proxy可以直接监听数组的变化

          # proxy和Object.defineProperty()区别

          Object.defineProperty只能劫持对象的属性，而Proxy是直接代理对象， Object.defineProperty对新增属性需要手动进行Observe Proxy支持13种拦截操作，这是defineProperty所不具有的

          # virtual DOM重构

          vue是模板templte, 2.0 是单个组件进行遍历, 3.0是比较动态节点。

          react的jsx和render function中一些可优化的信息丢失了,所以做了时间分片,在16ms的会更新一次

          # Vue composition API

          原有的写法，处理同个逻辑的方法分散在data method等各个地方。逻辑多了之后维护比较麻烦。

          组件从原本的选项配置变成了函数定义，组建逻辑结构更加灵活，逻辑复用

          • 更好的Typescript 类型推导支持
          • 逻辑复用, 组件从原本的选项配置变成了函数定义，组建逻辑结构更加灵活
          • tree-shaking ? 将 引用的 动态 import 包可以更下
          • 更好的压缩, 参数名可以压缩

          # setup(props, context)

          compostion api 统一入口，setup 函数会在 beforeCreate 之后、created 之前执行。 在 setup() 函数中无法访问到 this， context为上下文对象
