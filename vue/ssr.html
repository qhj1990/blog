      # SSR

      服务端渲染

      # 优点

      • 更好的SEO，搜索引擎爬虫能够查看完全渲染的页面
      • 更快的内容到到达时间，用户体验更好

      # 缺点

      • 一些生命周期钩子函数无法在服务端使用
      • 需要服务器渲染应用程序
      • 更多的服务器负载

      # Nuxt.js

      官网

      # 配置

      默认配置好大部分内容，可以在nuxt.config.js文件中修改配置

      export default {
/*
  ** 'spa': 没有服务器端渲染（只有客户端路由导航等）
  * 'universal': 同构应用程序（服务器端呈现+客户端路由导航等）
  */
  mode: 'universal',
  /*
  ** 网页html的header里的内容
  */
  head: {
    title: process.env.npm_package_name || '',
    meta: [
      { charset: 'utf-8' },
      { name: 'viewport', content: 'width=device-width, initial-scale=1' },
      { hid: 'description', name: 'description', content: process.env.npm_package_description || '' }
    ],
    link: [
      { rel: 'icon', type: 'image/x-icon', href: '/favicon.ico' }
    ]
  },
  /*
  ** 定制进度条颜色
  */
  loading: { color: '#fff' },
  /*
  ** 全局（所有页面均需引用的）样式文件、模块或第三方库。
  */
  css: [
  ],
  /*
  ** 需要在 根vue.js应用 实例化之前需要运行的 Javascript 插件。需要在plugins文件下创建plugin
  */
  plugins: [
  ],
  /*
  ** Nuxt.js dev-modules
  */
  buildModules: [
  ],
  /*
  ** Nuxt.js modules
  */
  modules: [
  ],
  /*
  ** 打包的配置信息
  */
  build: {
    /*
    ** 扩展webpack配置
    */
    extend (config, ctx) {
    }
  },
  // 配置 Nuxt.js 应用的根目录。
  rootDir: '',
  srcDir: '',
  vue: {
    config: {
      productionTip: true,
      devtools: false
    }
  },
  watch: ['~/custom/*.js']
}
