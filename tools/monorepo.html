      # Monorepo

      Monorepo是管理项目代码的方式，划分为多个模块，所有模块放在一个repo仓库。

      很多大型项目采用这种代码管理方式: babel, creat-react-app, create-router, vue-cli, dva, umi, webpack-cli

      # 优点

      • 代码结构清晰，利于团队协作

      # 缺点

      • 仓库项目庞大

      # Lerna

      Lerna用于管理具有多个程序包的JavaScript的工具。

      # 使用Lerna

      1. 全局安装lerna
      npm install -g lerna
      1
      1. 创建repo
      git init lerna-repo && cd lerna-repo
      1
      1. repo装加入lerna
      lerna init
      1

      # Lerna模式

      • Fixed 所有package共用一个版本号
      • Independent 每个包都有自己独立的版本号

      # Lerna 常用命令

      # 发布版本
lerna publish

# 查看版本
lerna version

# 安装所有依赖性并链接任何交叉依赖项
lerna bootstrap

# 列出当前 lerna 项目中的公共包
lerna list

# 检查上次release之后的哪些package更改
lerna changed

# 从上次发布以来，区分所有包和单个包
lerna diff [package]

# 在每个包中执行命令
lerna exec

# 在每一个含有该script的包执行npm script
lerna run [script]

# 创建一个新的lerna仓库或将现有的仓库升级到Lerna的当前版本。
lerna init --independent

# 安装依赖，支持交叉依赖
lerna add

# 删除项目中所有node_modules
lerna clean

# 导入
lerna import

# 软链
lerna link

# 创建子项目
lerna create

# 查看当前系统环境信息， OS、CPU、Node、Yarn、npm、Git等
lerna info

      # Yarn-workspaces

      yarn工作区，管理多个package时，允许只运行一次yarn install就能安装所有依赖。例如vue也在使用。

      # 使用

      package.json

      {
    "private": true,
    "workspaces": ["workspace-a", "workspace-b"]
}
