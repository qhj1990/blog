        # Rollup

        Rollup是JavaScript模块打包器，把小模块编译成整改复杂的大块代码，用于开发library和app。

        Rollup 还不支持一些特定的高级功能，尤其是用在构建一些应用程序的时候，特别是代码拆分和运行时态的动态导入 dynamic imports at runtime. 如果你的项目中更需要这些功能，那使用 Webpack可能更符合你的需求。

        • tree-shaking

        会分析代码中的import，会去除没有实际使用的代码

        # 配置文件

        rollup.config.js

        // rollup.config.js
export default {
  // 核心选项
  input,     // 必须
  external,
  plugins,

  // 额外选项
  onwarn,

  // danger zone
  acorn,
  context,
  moduleContext,
  legacy

  output: {  // 必须 (如果要输出多个，可以是一个数组)
    // 核心选项
    file,    // 必须
    format,  // 必须
    name,
    globals,

    // 额外选项
    paths,
    banner, // 字符串前置到bundle文件中
    footer, // 字符串后置到bundle文件中
    intro,
    outro,
    sourcemap,
    sourcemapFile,
    interop,

    // 高危选项
    exports,
    amd,
    indent
    strict
  },
};

        # input

        需要打包的文件

        # output

        # file

        要写入的文件。也可用于生成 sourcemaps

        # name

        iife/umd 包，同一页上的其他脚本可以访问它。

        # format

        • amd 异步模块定义，用于像RequireJS这样的模块加载器
        • cjs CommonJS, Node 模块化
        • esm ES6模块，在现代浏览器中可以通过 <script type=module> 标签引入
        • iife js立即执行函数
        • umd 通用模块定义，amd，cjs和iife为一体
        • system SystemJS格式
        • iife 立即执行

        # 插件

        1. rollup-plugin-json 加载json文件
        2. rollup-plugin-node-resolve
        3. rollup-plugin-commonjs
        4. rollup-plugin-babel
        5. rollup-plugin-typescript 处理ts文件
        6. rollup-plugin-vue 处理vue
        7. rollup-plugin-alias 为模块起别名的功能
        8. rollup-plugin-terser
        9. rollup-plugin-terser 压缩js代码，包括es6代码压缩

        # external

        外链

        # globals

        全局模块

        # watch

        检测到磁盘上单个模块已经改变，它会重新构建你的文件束bundle
