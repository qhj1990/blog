            # editorconfig

            帮助多个开发人员同个项目在不同的编辑器和IDE保持一致的编码样式.

            # 资源

            • 官网

            # 编辑器

            支持editorconfig的IDE和编辑器

            # 内置支持

            • WebStorm
            • PyCharm
            • VisualStudio
            • IntellijIDEA

            # 添加插件支持

            以下软件需要先安装插件才能正常使用.editorconfig

            • AppCode editorconfig
            • Atom editorconfig
            • Brackets editorconfig
            • eclipse editorconfig
            • phpStorm editorconfig
            • Sublime Text editorconfig
            • Vim editorconfig
            • VS Code EditorConfig for VS Code

            # 使用

            1. 在项目根目录新建.editorconfig文件。
            2. 编写属性
            # http://editorconfig.org

root = true

[*]
charset = utf-8
indent_style = space
indent_size = 2
end_of_line = lf
insert_final_newline = true
trim_trailing_whitespace = true

[*.md]
insert_final_newline = false
trim_trailing_whitespace = false

            # 属性

            • indent_style: 设置缩进风格，tab或者空格。tab是hard tabs，space为soft tabs
            • indent_size: 缩进的宽度，即列数，整数。如果indent_style为tab，则此属性默认为tab_width
            • tab_width: 设置tab的列数。默认是indent_size
            • end_of_line： 换行符，lf、cr和crlf
            • charset: 编码，latin1、utf-8、utf-8-bom、utf-16be和utf-16le，不建议使用utf-8-bom
            • trim_trailing_whitespace: 设为true表示会除去换行行首的任意空白字符
            • insert_final_newline: 设为true表明使文件以一个空白行结尾
            • root: 表明是最顶层的配置文件，发现设为true时，才会停止查找.editorconfig文件

            # 正则表达式

            • * 匹配除/之外的任意字符串
            • ** 匹配任意字符串
            • ? 匹配任意单个字符
            • [name] 匹配name字符
            • [!name] 匹配非name字符
            • {s1,s2,s3} 匹配任意给定的字符串,例如：a{1,2,3}b匹配a1b、a2b、a3b
            • {num1..num2} 匹配整数num1 -> 整数num2的闭区间,例如：{0..9}相当于[0-9]

            # 优先级

            每当打开文件时，插件会从文件所在目录开始并逐级向上寻找.editorconfig。

            只有在某个配置文件的root为true或已经到达根目录时，才会停止
